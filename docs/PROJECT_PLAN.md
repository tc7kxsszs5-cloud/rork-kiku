**Дата:** 17 февраля 2026

Цель этого документа — собрать в одном месте все 5 этапов: видение, требования, реализация, дорожная карта и конкретные действия «прямо сейчас».

---

## 1) Идея и видение

**KIDS by KIKU** — AI‑платформа для защиты детей в цифровой среде, совмещающая **проактивную защиту** и **обучение безопасному поведению**.

Ключевой фокус:
- Предотвращать угрозы до того, как ребенок их увидит.
- Обучать безопасному поведению в игровой/доступной форме.
- Давать родителям прозрачный контроль и аналитику.
- Быть кросс‑платформенным стандартом (iOS/Android/веб).

---

## 2) Требования (что нужно для реализации)

### Функциональные требования (MVP)
- AI‑модерация контента (тексты/медиа) до отображения ребенку.
- Родительский контроль и аналитика.
- Образовательные сценарии безопасного поведения.
- SOS‑функции с геолокацией и оповещением.
- Авторизация и роли (родитель/ребенок).

### Нефункциональные требования
- Безопасность и соответствие нормам (COPPA/GDPR‑K).
- Мониторинг ошибок и метрик (Sentry).
- Масштабируемость API и базы данных.
- Надежная инфраструктура и защищенное хранение секретов.

### Инфраструктура и зависимости
- Backend API (Hono + tRPC).
- PostgreSQL (рекомендуется Supabase).
- Push‑уведомления (FCM/APNS).
- OpenAI API key для модерации.
- Sentry DSN для мониторинга.

---

## 3) Реализация в коде (текущее состояние)

### Основной стек
- **Frontend:** Expo React Native + Expo Router.
- **Backend:** Hono + tRPC (`backend/`).
- **База:** PostgreSQL (`backend/schema.sql`).
- **Мониторинг:** Sentry (`utils/sentry.ts`).
- **Деплой:** Vercel (backend), EAS (iOS/Android).

### Текущее состояние (по отчету от 1 февраля 2026)
- Код подготовлен к продакшн с рекомендациями.
- TypeScript и ESLint без критических ошибок.
- Тесты в целом проходят, есть незначимые warnings.
- Остались инфраструктурные задачи: backend, база, env, push, Sentry.

## 4) Дорожная карта

### Этап A — Инфраструктурная готовность

### Этап C — Production релиз

## 5) Что делать прямо сейчас (приоритетные действия)

1. Развернуть backend (`backend/`) и проверить доступность URL.
2. Настроить PostgreSQL и применить миграции (`backend/schema.sql`).
3. Заполнить `.env.production` и добавить EAS secrets:
   - `OPENAI_API_KEY`
   - `DATABASE_URL`
   - `JWT_SECRET`
   - `SENTRY_DSN`
4. Настроить push‑уведомления (FCM/APNS) и `projectId`.
5. Провести короткий smoke‑test через Expo.
6. Запустить production build через EAS.
7. Подготовить и опубликовать Privacy Policy и Terms.
