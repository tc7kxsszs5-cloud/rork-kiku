## Рекомендация: MIT License

### Причины выбора MIT:
1. **Простота**: Легко понять, минимальные ограничения
2. **Flexibility**: Позволяет коммерческое использование
3. **Популярность**: Один из самых популярных OSS licenses (Node.js, React)
4. **Business-friendly**: Совместим с proprietary commercial software

### Что разрешает MIT:
- ✅ Коммерческое использование
- ✅ Modification
- ✅ Distribution
- ✅ Private use

### Что требует MIT:
- Включить copyright notice и license text в копиях

### Альтернативы:

**Apache 2.0**:
- Более детальный, explicit patent grant
- Хороший выбор если patent protection важна

**AGPL 3.0**:
- Copyleft (требует раскрытие изменений)
- **НЕ рекомендуется** для commercial startup

### Итоговая рекомендация:
**MIT License** для Rork-Kiku, если планируется open-source части кода (например, SDK, libraries).

**Proprietary License** для core business logic (модерация, ML models) — не раскрывать публично.
