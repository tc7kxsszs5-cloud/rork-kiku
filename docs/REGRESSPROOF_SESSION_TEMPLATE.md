# RegressProof Session Template

Use this template at the end of each meaningful work session.

Store session notes under `docs/sessions/` using a filename like:

- `YYYY-MM-DD-session.md`

## Template

```md
# RegressProof Session Note

**Date:** YYYY-MM-DD
**Focus:** short description of the session goal
**Status:** completed | partial | blocked

## Completed

- what was finished
- what was validated

## Changed Files

- file paths touched in this session

## Decisions

- decisions made during the session

## Open Questions

- unresolved risks, blockers, or follow-ups

## Next Best Step

- one concrete next action for the next session
```

## Session Closeout Rule

At the end of a session:

1. Add or update one session note in `docs/sessions/`.
2. Update `docs/REGRESSPROOF_DECISION_LOG.md` if a durable decision was made.
3. Update `docs/REGRESSPROOF_IMPLEMENTATION_PLAN.md` or `docs/REGRESSPROOF_MVP_TASK_BREAKDOWN.md` if priorities changed.
4. Keep `docs/REGRESSPROOF_WORKFLOW_MEMORY.md` aligned with the latest practical next step.
