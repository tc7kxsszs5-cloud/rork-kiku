# Чеклист документов для Data Room

## Обзор

Data Room — это безопасное виртуальное хранилище документов для due diligence инвесторов. Этот чеклист содержит полный список документов, которые должны быть подготовлены для потенциальных инвесторов (Seed, Series A и далее).

⚠️ **Конфиденциальность**: Доступ к Data Room предоставляется только после подписания NDA (Non-Disclosure Agreement).

---

## 1. Корпоративные документы (Corporate)

- [ ] Advisory Board composition (если есть)

### 1.3 Funding History

- [ ] Side letters (если были специальные условия для инвесторов)

- [ ] Voting agreements

### 1.4 Corporate Governance
- [ ] Board meeting minutes (последние 12-24 месяцев)
- [ ] Shareholder meeting minutes
- [ ] Written consents
- [ ] Committee charters (если есть комитеты: audit, compensation)

### Tools для Data Room:

**Recommended**:
1. **Docsend** — популярный, analytics, NDA-gating
2. **Google Drive** с controlled access (дешевле)
3. **Box** — enterprise-grade security
4. **Dropbox Business** — знакомый интерфейс
5. **Intralinks** — для крупных deals (дорого)

**Best practices**:
- Версионирование документов (v1, v2, etc.)
- Watermarking на confidential docs
- Track кто что открывал (analytics)
- NDA перед доступом
- Regular updates (keep it current)

## Timeline для подготовки Data Room

### Pre-Fundraising (3 months before)
- [ ] Audit существующих документов
- [ ] Identify gaps
- [ ] Собрать недостающие документы
- [ ] Organize в структуру
- [ ] Legal review (чувствительные документы)

### Fundraising Active (during)
- [ ] Setup Data Room tool
- [ ] Upload все документы
- [ ] Test access (dry run с advisor)
- [ ] NDA template ready
- [ ] Prepare Q&A doc (FAQ на типичные вопросы)

### During DD (due diligence)
- [ ] Grant access к заинтересованным инвесторам (после NDA)
- [ ] Monitor доступ (кто что смотрит)
- [ ] Respond to questions быстро (< 24h)
- [ ] Update documents as needed

## Red Flags для инвесторов

Избегайте этих распространенных ошибок:

❌ **Incomplete Cap Table** — неточная или устаревшая
❌ **No financial projections** — показывает lack of planning
❌ **Messy corporate records** — missing board minutes, signatures
❌ **IP not assigned** — employees не подписали IP assignment
❌ **Outstanding legal issues** — неразрешенные litigation
❌ **Compliance gaps** — COPPA/GDPR нарушения
❌ **High employee turnover** — без explanation
❌ **Unclear use of funds** — "just give us money"

✅ **What investors want to see**:
- Организованность и профессионализм
- Transparency (честность о challenges)
- Clear plan (куда идут деньги, что достигнем)
- Strong team (retention, experience)
- Customer traction (metrics, testimonials)

## Конфиденциальность и безопасность

### Sensitive Information

**НЕ включать** в Data Room (предоставлять только на поздних стадиях DD):
- Полные списки customers с контактами (только aggregated metrics)
- Sensitive employee info (SSN, personal details)
- Source code (только architecture docs)
- Passwords, API keys, secrets
- Personal communications (emails, Slack logs)

**Redact** sensitive info:
- Customer names (если не public) → "Fortune 500 Company A"
- Employee salaries в early stages → показывать ranges
- Specific security vulnerabilities → general security posture

### Access Control

- **NDA** обязательна перед доступом
- **Tiered access**: Разный уровень для разных инвесторов
  - Tier 1 (Early interest): Pitch deck, one-pager
  - Tier 2 (Serious interest): Financials, product docs
  - Tier 3 (Term sheet signed): Full data room
- **Expiring access**: Links с expiration (90 days)
- **Watermarking**: На особо чувствительных документах
