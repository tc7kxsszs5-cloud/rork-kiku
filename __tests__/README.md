# Детерминированные модульные тесты

## Принципы

Все тесты в этой директории должны быть **детерминированными**:

- ✅ **Только чистые функции** - без побочных эффектов
- ✅ **Нет интеграций** - без внешних зависимостей
- ✅ **Нет асинхронности** - все синхронные операции
- ✅ **Нет сложных моков** - только простые моки (например, Platform.OS)
- ✅ **Детерминированность** - одинаковый вход = одинаковый выход

## Что НЕ тестируем

- ❌ Асинхронные функции (async/await)
- ❌ Функции с интеграциями (API, база данных, файловая система)
- ❌ Функции, требующие сложных моков
- ❌ Компоненты React (это интеграционные тесты)
- ❌ Context providers (это интеграционные тесты)

## Что тестируем

- ✅ Чистые функции-утилиты
- ✅ Функции преобразования данных
- ✅ Функции вычислений без побочных эффектов
- ✅ Константы и конфигурации

## Структура

```
__tests__/unit/
  utils/
    soundNotifications.test.ts  # Тесты чистых функций sound notifications
    cursorStyles.test.ts        # Тесты констант cursor styles
```

## Запуск

```bash
# Все unit тесты
bun test --testPathPattern=unit

# Конкретный файл
bun test soundNotifications.test.ts
```

## CI/CD

Тесты запускаются автоматически в GitHub Actions при каждом push/PR.
